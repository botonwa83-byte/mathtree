## 一、🔴 阻断级（提交前必须解决）

### 1. KaTeX 本地化（离线渲染）—— 当前最高风险
- 现状：`FormulaView.swift` 从 `cdn.jsdelivr.net` 远程加载 KaTeX 的 css/js。
- 风险：**离线或弱网时所有公式无法显示**；审核员常在受限网络测试 → 大概率被拒，且核心体验不可用。
- 待办：
  - [ ] 将 `katex.min.css`、`katex.min.js` 及字体文件(`fonts/`)打包进 App Bundle（Resources）。
  - [ ] `loadHTMLString` 改用 `baseURL` 指向本地 Bundle，或用 `WKWebView.loadFileURL`。
  - [ ] 验证：开飞行模式后所有公式正常渲染。
  - [ ] 标注 KaTeX 的 MIT 许可（见第六节）。

### 2. 隐私清单 PrivacyInfo.xcprivacy（缺失）
- 现状：无 `.xcprivacy` 文件；App 在 PracticeManager/StreakManager/ProblemMasteryManager/ReviewScheduler 使用 `UserDefaults`（属于 Apple "Required Reason API"）。
- 待办：
  - [ ] 新增 `PrivacyInfo.xcprivacy` 并加入 target。
  - [ ] 声明 UserDefaults 使用原因码 `CA92.1`（仅本 App 读写自身数据）。
  - [ ] `NSPrivacyTracking = false`；`NSPrivacyCollectedDataTypes = []`（本 App 不收集任何数据）。

### 3. App 图标透明通道（会被拒）
- 现状：`AppIcon-1024@1x.png` 为 **RGBA（含 alpha）**。App Store 商店图标**不允许透明/alpha 通道**。
- 待办：
  - [ ] 将 1024 及各尺寸图标导出为**无 alpha 的 RGB PNG**（背景不透明）。
  - [ ] 图标无圆角（系统自动加圆角）、无穿透。

### 4. Release 构建与 Archive 可成功
- [ ] 用 **Release** 配置 Archive 成功（非 Debug）。
- [ ] 真机（非模拟器）安装运行通过。
- [ ] Xcode Organizer → Validate App 通过（无缺失图标/签名/Info 项报错）。

## 四·补、🟡 iPad 适配专项（device family=1,2 已声明支持，必做）

> 既然声明支持 iPad，App Store 会**在 iPad 上审核**，布局破裂/拉伸/崩溃会被拒，且**必须提供 iPad 截图**。
> 现状盘点：`TabView` + 每个 Tab `NavigationStack`；`FormulaUniverseView`、`BattlefieldView` 已用 `@Environment(\.horizontalSizeClass)` 做 `isIPad` 分支；**其余视图（CommandCenter / ErrorBook / Heroes / MysteryRoom / Review / DailyStrike / FormulaDetail）未见 iPad 适配**。

### 必做
- [ ] **全视图 iPad 走查**（竖屏+横屏，11" 与 12.9"）：CommandCenter、ErrorBook、Heroes、MysteryRoom、Review、DailyStrike、FormulaDetail 布局不破、不空旷、不拉伸。
- [ ] **限制正文可读宽度**：长文本/卡片用 `.frame(maxWidth: 600~760)` 居中，避免文字/题干横跨整屏（大屏阅读体验差是高频扣分点）。
- [ ] **自适应网格列数**：公式宇宙、英雄录、题目/错题列表用 `LazyVGrid(adaptive)` 或按 `sizeClass` 增加列数，充分利用大屏。
- [ ] **统一 isIPad 工具**：把已散落在两个文件的 `isIPad` 抽成共用扩展，全 App 一致使用。
- [ ] **KaTeX 大屏渲染**：iPad 上公式字号、换行、`FormulaView` 高度自适应正常（WKWebView 内容宽度随容器变化）。
- [ ] **iPad 多任务**：Split View / Slide Over 下不崩、不错位（如不想支持，可在 Info 设 `UIRequiresFullScreen=YES`）。
- [ ] **导航体验**（可选进阶）：iPad 上考虑用 `NavigationSplitView`（侧栏+详情）替代纯 `NavigationStack`，更符合 iPad 习惯。
- [ ] **真机+模拟器验证**：iPad 模拟器（11"、12.9"）与（若有）真机各跑一遍，顺便截图供商店使用。

### 决策点
- [ ] 若短期无力做好 iPad → 改 `TARGETED_DEVICE_FAMILY = 1`（仅 iPhone），可省去 iPad 截图与适配工作（**但会失去 iPad 用户**）。建议：本 App 内容适合 iPad 大屏阅读公式，**优先做好 iPad 适配**。

- [ ] iPad 崩溃或布局破裂（声明支持 iPad 却体验差）。
- [ ] 元数据含其他平台名/价格/外部购买引导。
